# AI_INTEGRATION_PLAN.md

## 1. 목적

이 문서는 대화코칭어플에 실제 AI API를 연결할 때 어떤 순서로 진행해야 하는지 정리한다.

현재 단계에서는 API를 연결하지 않는다.
먼저 안전한 연결 구조를 정한다.

## 2. 현재 구조

현재 앱은 아래 3개 파일로 작동한다.

- index.html
- style.css
- app.js

이 구조는 브라우저에서 직접 실행되는 정적 웹앱이다.

장점:

- 단순하다.
- 실행이 쉽다.
- 비개발자도 index.html을 열어 확인할 수 있다.

단점:

- API 키를 숨길 수 없다.
- 실제 AI API를 직접 호출하면 보안 문제가 생긴다.
- 사용량 제한이나 에러 처리가 어렵다.

## 3. AI 연결 기본 방향

실제 AI API 연결은 아래 구조로 진행해야 한다.

사용자 브라우저  
→ 우리 서버 또는 서버리스 함수  
→ AI API  
→ 우리 서버 또는 서버리스 함수  
→ 사용자 브라우저

즉, 브라우저가 AI API를 직접 호출하지 않는다.

## 4. 추천 연결 방식

초기에는 아래 방식이 적합하다.

1. 프론트엔드는 현재 HTML/CSS/JS 구조를 최대한 유지한다.
2. AI 호출만 별도 서버 또는 서버리스 함수로 분리한다.
3. API 키는 서버 쪽 환경 변수에 저장한다.
4. app.js는 우리 서버의 안전한 endpoint만 호출한다.
5. 서버는 AI_OUTPUT_SPEC.md 형식에 맞는 결과만 반환한다.

## 5. 아직 하지 않을 것

초기 AI 연결 전에는 아래를 하지 않는다.

- 로그인
- 결제
- 사용자 DB 저장
- 대화 기록 저장
- 모바일 앱 변환
- 관리자 페이지
- 복잡한 프롬프트 관리 도구
- 다중 모델 선택 기능
- 사용자별 사용량 과금

## 6. AI 요청에 포함할 입력값

AI에게 전달할 입력값은 아래로 제한한다.

```json
{
  "situation": "사용자가 입력한 현재 상황",
  "other_message": "상대방의 최근 메시지",
  "user_intent": "사용자의 답장 의도"
}
```

불필요한 개인정보를 요구하지 않는다.

## 7. AI에게 요구할 출력값

AI는 반드시 AI_OUTPUT_SPEC.md의 구조로 응답해야 한다.

사용자 화면에는 아래만 보여준다.

- strategy_summary
- recommendations

화면에 보여주지 않을 것:

- internal_evaluation
- failure_codes
- 평가 점수
- 개발용 판단 근거
- 프롬프트 원문

## 8. 실패 시 fallback 정책

AI API 호출이 실패하면 사용자는 복잡한 오류 메시지를 보면 안 된다.

사용자에게는 아래처럼 안내한다.

“지금은 답장 방향을 불러오지 못했어요. 잠시 후 다시 시도해 주세요.”

가능하면 기존 로컬 예시 응답을 fallback으로 사용할 수 있다.

## 9. 응답 검증 정책

AI 응답을 받은 뒤 아래를 확인한다.

1. strategy_summary가 있는가
2. recommendations가 정확히 3개인가
3. 각 recommendation에 label, message, reason이 있는가
4. internal_evaluation이 있어도 화면에는 숨겨지는가
5. 금지 문장 유형이 포함되어 있지 않은가
6. message가 너무 길지 않은가
7. 조종, 밀당, 질투 유발 표현이 없는가

## 10. 다음 구현 단계 후보

AI 연결 전 다음 단계는 아래 순서로 진행한다.

1. API 연결 없이 현재 로컬 MVP 최종 검수
2. TEST_SCENARIOS.md 기준으로 결과 품질 확인
3. AI 연결용 서버 구조 선택
4. API 키 보관 방식 결정
5. 서버 또는 서버리스 함수 설계
6. app.js에서 서버 endpoint를 호출하도록 변경
7. AI 응답 검증 로직 추가
8. fallback 처리 추가
9. 실제 AI API 연결 테스트

## 11. 비개발자 보호 원칙

사용자는 코딩을 모른다.

따라서 이후 단계에서도 아래 원칙을 지킨다.

- 실행 전 반드시 설명한다.
- 터미널 명령은 최소화한다.
- 사용자가 눌러야 할 버튼을 명확히 말한다.
- API 키를 코드에 붙여넣게 하지 않는다.
- 위험한 작업은 먼저 멈추고 설명한다.
- 파일을 만들 때는 어떤 파일이 왜 필요한지 설명한다.
