# MVP_IMPLEMENTATION_PLAN.md

## 1. MVP 목표

대화코칭어플의 첫 MVP는 사용자가 상황 설명, 상대방 메시지, 자신의 의도를 입력하면 strategy_summary와 recommendations 3개를 보여주는 아주 단순한 웹앱이다.

초기 목표는 완성형 서비스가 아니라, 제품 철학이 실제 화면과 결과 구조로 작동하는지 확인하는 것이다.

## 2. MVP 기능 범위

MVP에 포함할 기능은 아래로 제한한다.

1. 현재 상황 입력
2. 상대방의 최근 메시지 입력
3. 사용자의 의도 입력 또는 선택
4. strategy_summary 출력
5. recommendations 3개 출력
   - 안전한 답장
   - 조금 더 다정한 답장
   - 선을 긋는 답장
6. internal evaluation 구조는 내부 코드상으로만 유지
7. 사용자가 보게 되는 화면에는 strategy_summary와 recommendations만 노출

## 3. 첫 화면 구성

첫 화면에는 아래 영역만 둔다.

1. 앱 제목
2. 짧은 설명 문구
3. 현재 상황 입력창
4. 상대방 메시지 입력창
5. 내 의도 선택 또는 입력 영역
6. 결과 생성 버튼
7. strategy_summary 결과 영역
8. recommendations 결과 영역

초기 화면은 복잡하게 만들지 않는다.
카드, 탭, 로그인, 메뉴, 사이드바는 만들지 않는다.

## 4. 사용자 입력 항목

사용자가 입력해야 하는 항목은 아래 3개다.

### 현재 상황

예:
- 썸을 타는 중인데 답장이 느려졌다.
- 친구에게 서운한 일이 있었다.
- 상대가 애매하게 답해서 어떻게 답해야 할지 모르겠다.

### 상대방의 최근 메시지

예:
- “요즘 좀 바빠서 답장이 늦었어.”
- “그냥 편하게 생각해.”
- “나중에 얘기하자.”

### 사용자의 의도

초기에는 아래 선택지를 둔다.

1. 부담 없이 이어가고 싶다
2. 조금 더 다정하게 표현하고 싶다
3. 서운함을 차분히 말하고 싶다
4. 선을 긋고 싶다
5. 잘 모르겠다

## 5. 앱이 보여줘야 하는 결과

사용자에게 보여주는 결과는 아래 2개만 둔다.

### strategy_summary

- 현재 상황 요약
- 대화에서 조심해야 할 점
- 추천되는 답장 방향

### recommendations

답장 후보 3개를 보여준다.

1. 안전한 답장
2. 조금 더 다정한 답장
3. 선을 긋는 답장

각 추천 문장은 너무 길지 않아야 한다.
각 문장은 실제 카카오톡이나 문자로 보낼 수 있을 정도로 자연스러워야 한다.
사용자에게 조종, 밀당, 시험, 불안 유발을 권하지 않는다.

## 6. 내부 evaluation 구조

화면에는 노출하지 않지만, 내부적으로는 아래 평가 구조를 염두에 둔다.

- burden_level
- naturalness
- actual_use_likelihood
- relationship_risk
- emotional_overreaction_risk
- manipulation_risk
- regret_risk
- failure_codes

failure_codes는 향후 F01~F15 구조를 붙일 수 있도록 배열 형태로 둔다.

## 7. 아직 만들지 않을 기능

초기 MVP에서는 아래 기능을 만들지 않는다.

- 로그인
- 회원가입
- 결제
- DB 저장
- 사용자 기록 저장
- 관리자 페이지
- 배포
- 모바일 앱 전환
- 카카오톡 연동
- 실제 AI API 연결
- 복잡한 프롬프트 관리 화면
- 사용자 계정 관리
- 통계 화면

## 8. 추천 기술 방향

비개발자 사용자가 다루기 쉬운 단순한 웹앱으로 시작한다.

초기에는 복잡한 백엔드나 DB 없이, 프론트엔드 중심의 단순 화면부터 만든다.
실제 AI API 연결은 나중 단계로 미룬다.
처음에는 예시 응답 또는 로컬 함수 형태로 결과 구조를 확인한다.

## 9. 개발 순서

개발 순서는 아래처럼 진행한다.

1. 문서 확정
2. 화면 설계 문서 작성
3. 파일 구조 문서 작성
4. 정적 화면 코드 작성
5. 입력값을 받아 결과를 보여주는 로컬 로직 작성
6. 결과 구조 검수
7. UI 문구 조정
8. 나중에 AI API 연결 검토
