# UI_WIREFRAME.md

## 1. 화면 목표

첫 화면의 목표는 사용자가 복잡한 설정 없이 자신의 대화 상황을 입력하고, 바로 strategy_summary와 recommendations를 확인하게 하는 것이다.

이 화면은 완성형 서비스 화면이 아니라 MVP 검증용 화면이다.

## 2. 화면 구성 순서

첫 화면은 위에서 아래로 아래 순서로 구성한다.

1. 앱 제목
2. 짧은 설명 문구
3. 현재 상황 입력 영역
4. 상대방 메시지 입력 영역
5. 내 의도 선택 영역
6. 결과 생성 버튼
7. strategy_summary 결과 영역
8. recommendations 결과 영역

## 3. 앱 제목 영역

화면 상단에는 아래 제목을 표시한다.

대화코칭어플

제목 아래에는 아래 설명 문구를 표시한다.

보내기 전에 한 번 더 생각하게 도와주는 대화 코칭 도구

## 4. 현재 상황 입력 영역

라벨:

현재 어떤 상황인가요?

placeholder 예시:

예: 썸을 타는 중인데 상대 답장이 예전보다 느려졌어요.

입력 방식:

여러 줄 입력창

## 5. 상대방 메시지 입력 영역

라벨:

상대방이 뭐라고 보냈나요?

placeholder 예시:

예: 요즘 좀 바빠서 답장이 늦었어.

입력 방식:

여러 줄 입력창

## 6. 내 의도 선택 영역

라벨:

나는 어떻게 답하고 싶나요?

선택지는 아래로 둔다.

1. 부담 없이 이어가고 싶다
2. 조금 더 다정하게 표현하고 싶다
3. 서운함을 차분히 말하고 싶다
4. 선을 긋고 싶다
5. 잘 모르겠다

## 7. 결과 생성 버튼

버튼 문구:

답장 방향 보기

주의:

버튼은 실제 AI API를 호출하지 않는다.
초기 MVP에서는 입력값을 바탕으로 로컬 예시 결과를 보여주는 역할만 한다.

## 8. strategy_summary 결과 영역

제목:

상황 정리

보여줄 내용:

- 현재 상황 요약
- 조심해야 할 점
- 추천 답장 방향

## 9. recommendations 결과 영역

제목:

답장 후보

답장 후보는 3개 카드로 보여준다.

1. 안전한 답장
2. 조금 더 다정한 답장
3. 선을 긋는 답장

각 카드에는 아래를 포함한다.

- 답장 유형 이름
- 실제 보낼 수 있는 문장
- 왜 이 답장이 적절한지 짧은 설명

## 10. 화면에서 숨길 것

아래 정보는 화면에 노출하지 않는다.

- internal evaluation
- failure_codes
- burden_level 점수
- manipulation_risk 점수
- regret_risk 점수
- 개발용 로그
- API 키
- 프롬프트 원문
