# AI 서버 안전 규칙

## 1. 목적

이 문서는 대화코칭어플에 AI API를 연결할 때 서버와 API 키를 안전하게 다루기 위한 규칙이다.

현재 사용자는 비개발자이므로 API 키를 코드에 직접 붙여넣게 하면 안 된다.

## 2. 절대 금지

아래는 절대 금지한다.

- API 키를 app.js에 넣기
- API 키를 index.html에 넣기
- API 키를 브라우저에서 볼 수 있는 코드에 넣기
- API 키가 들어간 파일을 GitHub에 올리기
- API 키를 채팅창에 그대로 노출하기
- AI 응답 원문 전체를 사용자에게 그대로 노출하기
- internal_evaluation을 화면에 보여주기
- 사용자의 민감한 개인정보를 불필요하게 AI에 보내기

## 3. API 키 보관 원칙

API 키는 반드시 서버 쪽 환경 변수에 보관한다.

**허용 위치**

- 서버 환경 변수
- 서버리스 함수 환경 변수
- 배포 플랫폼의 Secret
- 배포 플랫폼의 Environment Variable

**금지 위치**

- index.html
- style.css
- app.js
- README.md
- 문서 파일
- 브라우저 localStorage
- 브라우저 console
- 공개 저장소

## 4. 서버가 해야 할 일

서버 또는 서버리스 함수는 아래 역할을 한다.

1. 프론트엔드에서 사용자 입력을 받는다.
2. 입력값을 간단히 검증한다.
3. API 키를 사용해 AI API를 호출한다.
4. AI 응답을 정해진 구조로 받는다.
5. 응답 구조를 검증한다.
6. 금지 표현을 점검한다.
7. 사용자에게 필요한 필드만 반환한다.

## 5. 프론트엔드가 해야 할 일

프론트엔드는 아래 역할만 한다.

1. 사용자 입력을 받는다.
2. 서버 endpoint에 요청한다.
3. 서버가 반환한 결과를 화면에 보여준다.
4. internal_evaluation은 화면에 표시하지 않는다.
5. 오류가 나면 쉬운 안내 문구를 보여준다.

## 6. 오류 처리

AI 호출이 실패했을 때 사용자에게 아래처럼 안내한다.

지금은 답장 후보를 불러오지 못했어요. 잠시 후 다시 시도해 주세요.

개발용 오류 메시지, API 오류 코드, stack trace는 사용자에게 보여주지 않는다.

## 7. 개인정보 최소화

초기 버전에서는 아래 정보만 AI에 보낸다.

- 현재 상황
- 상대방 메시지
- 사용자의 관계 목표

**아래는 요구하지 않는다.**

- 실명
- 전화번호
- 카카오톡 ID
- 주소
- 민감한 개인정보
- 전체 대화 기록

## 8. 응답 필터링

서버는 AI 응답에서 아래를 걸러야 한다.

- 질투 유발
- 일부러 답장 늦추기
- 상대 시험하기
- 보복성 답장
- 상대를 불안하게 만들기
- 과도한 장문
- AI 예시문 느낌이 강한 문장
- internal_evaluation 노출

## 9. 비개발자 보호

사용자는 코딩을 모른다.

따라서 이후 구현 단계에서는 아래 원칙을 따른다.

- 실행 전 먼저 설명한다.
- 터미널 명령은 꼭 필요한 경우에만 사용한다.
- 사용자가 눌러야 할 버튼을 명확히 말한다.
- API 키를 어디에 넣는지 매우 구체적으로 안내한다.
- 코드에 API 키를 직접 붙여넣지 않게 한다.
- Run 버튼이 생길 수 있는 작업은 먼저 멈추고 설명한다.

## 10. 최종 원칙

AI 연결은 품질 개선을 위한 필수 단계다.

하지만 보안 구조 없이 AI를 연결하면 안 된다.

API 키는 절대 브라우저 코드에 들어가면 안 된다.
