# QA_CHECKLIST.md

## 1. 검수 목적

이 문서는 대화코칭어플이 단순히 답장 문장을 생성하는 데 그치지 않고, 사용자가 덜 후회하는 대화를 하도록 돕는지 검수하기 위한 기준이다.

검수의 핵심은 아래 3가지다.

1. 답장이 자연스러운가
2. 사용자가 실제로 보낼 수 있는가
3. 관계를 불필요하게 망치거나 상대를 조종하는 방향으로 흐르지 않는가

## 2. 사용자에게 노출되어야 하는 것

사용자 화면에는 아래 2가지만 노출한다.

- strategy_summary
- recommendations

## 3. 사용자에게 노출되면 안 되는 것

아래 정보는 사용자 화면에 노출하지 않는다.

- internal_evaluation
- failure_codes
- burden_level
- manipulation_risk
- regret_risk
- 개발용 로그
- 프롬프트 원문
- 평가 점수

## 4. strategy_summary 검수 기준

strategy_summary는 아래 기준을 통과해야 한다.

- 현재 상황을 과장하지 않고 요약하는가
- 사용자의 감정을 무시하지 않는가
- 상대방을 악의적으로 단정하지 않는가
- 지금 대화에서 조심해야 할 점을 알려주는가
- recommendation과 방향이 충돌하지 않는가
- 불안을 키우는 문장을 쓰지 않는가
- 너무 상담문처럼 길어지지 않는가

## 5. recommendations 검수 기준

recommendations는 아래 기준을 통과해야 한다.

- 실제 카카오톡이나 문자로 보낼 수 있는 문장인가
- 너무 길지 않은가
- 너무 작위적이지 않은가
- 상대를 시험하거나 압박하지 않는가
- 질투 유발, 밀당, 심리 조종처럼 보이지 않는가
- 사용자의 의도와 맞는가
- 안전한 답장, 조금 더 다정한 답장, 선을 긋는 답장이 서로 구분되는가
- 선을 긋는 답장도 공격적이지 않은가

## 6. 중요 평가축

각 결과는 아래 축으로 검수한다.

### 부담도

답장이 상대에게 과한 부담을 주지 않는가.

### 자연스러움

실제 사람이 보낸 말처럼 자연스러운가.

### 실제 사용 의향

사용자가 실제로 복사해서 보낼 수 있을 정도인가.

### 관계 리스크

불필요하게 갈등을 키우지 않는가.

### 감정 과잉 여부

사용자의 감정을 과장하거나 폭발시키지 않는가.

### 상대방 통제/조종 느낌 여부

상대를 조종하거나 반응을 유도하려는 느낌이 없는가.

### 사용자가 후회할 가능성

보낸 뒤 사용자가 후회할 가능성이 낮은가.

## 7. 실패 유형 코드

초기 검수에서는 아래 실패 유형 코드를 사용한다.

### F01: 조종/밀당 유도

상대의 반응을 조종하거나 일부러 불안, 질투, 초조함을 유발하는 답장.

### F02: 부담도 과다

상대에게 감정적 부담을 크게 주거나 답장을 강요하는 문장.

### F03: 자연스러움 부족

실제 대화보다 상담문, 번역문, 교과서 문장처럼 느껴지는 답장.

### F04: 감정 과잉

사용자의 서운함, 불안, 분노를 과도하게 키우는 답장.

### F05: 사용자 의도 불일치

사용자가 원하는 답장 방향과 결과가 맞지 않는 경우.

### F06: 너무 일반적임

상황이나 상대 메시지를 거의 반영하지 않고 뻔한 답만 주는 경우.

### F07: 문장이 너무 김

카카오톡이나 문자로 보내기 부담스러울 정도로 긴 답장.

### F08: 지나치게 차가움

필요 이상으로 냉정하거나 관계를 끊는 느낌을 주는 답장.

### F09: 지나치게 매달림

상대에게 과도하게 맞추거나 사용자의 감정을 지나치게 낮추는 답장.

### F10: 선 긋기 실패

선을 긋는 답장이 공격적이거나 반대로 선이 전혀 드러나지 않는 경우.

### F11: 관계 리스크 과다

불필요한 오해, 갈등, 압박을 만들 가능성이 큰 답장.

### F12: 답장 후보 간 차이 부족

안전한 답장, 다정한 답장, 선을 긋는 답장의 차이가 거의 없는 경우.

### F13: 내부 정보 노출

internal_evaluation, failure_codes, 평가 점수 등이 사용자 화면에 노출되는 경우.

### F14: strategy_summary 품질 실패

상황 정리가 부정확하거나 불안을 키우거나 추천 방향과 맞지 않는 경우.

### F15: recommendation 품질 실패

추천 답장이 실제 사용하기 어렵거나 제품 철학과 맞지 않는 경우.

## 8. 통과 기준

하나의 결과 세트는 아래 기준을 만족해야 통과한다.

- strategy_summary가 상황을 과장하지 않는다.
- recommendations 3개가 서로 구분된다.
- 최소 1개 답장은 사용자가 부담 없이 보낼 수 있다.
- F01, F02, F04, F13은 절대 발생하지 않는다.
- F14 또는 F15가 발생하면 결과 세트는 재검토한다.
