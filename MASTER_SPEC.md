# MASTER_SPEC.md

## 제품명

대화코칭어플

## 제품 목적

대화코칭어플은 사용자가 연애, 썸, 친구, 갈등, 서운함, 텐션 저하, 관계 애매함 등의 상황에서 상대에게 보낼 말을 더 안전하고 자연스럽게 정리하도록 돕는 대화 코칭 앱이다.

## 핵심 철학

이 앱은 상대를 조종하거나 밀당 기술을 가르치는 앱이 아니다.

이 앱은 사용자가 감정적으로 과잉 반응하거나, 너무 차갑게 굴거나, 불필요하게 관계를 망치는 메시지를 보내기 전에 한 번 더 생각하게 만드는 코칭 도구다.

좋은 답장은 상대를 이기기 위한 말이 아니라, 관계의 맥락과 사용자의 감정을 모두 덜 해치도록 조율된 말이어야 한다.

## 핵심 사용자

- 상대에게 뭐라고 답장해야 할지 모르는 사람
- 감정적으로 답장을 보내고 후회한 경험이 있는 사람
- 너무 차갑거나 너무 매달리는 메시지를 피하고 싶은 사람
- 썸, 연애, 친구 관계, 갈등 상황에서 자연스러운 말을 찾고 싶은 사람
- 자신의 감정은 지키되 관계를 불필요하게 망치고 싶지 않은 사람

## 핵심 사용 상황

1. 호감 탐색
2. 텐션 하락
3. 서운함/갈등
4. 관계 애매함
5. 답장 타이밍 고민
6. 부담스럽지 않게 마음 표현하기
7. 선을 긋되 관계를 망치지 않기

## 핵심 출력 구조

### 1. strategy_summary

사용자가 지금 어떤 상황에 있는지 요약한다.
현재 대화에서 가장 중요한 위험과 방향을 설명한다.
recommendations와 분리해서 평가할 수 있어야 한다.

### 2. recommendations

사용자가 실제로 보낼 수 있는 답장 후보를 제공한다.
초기 MVP에서는 3개를 제공한다.

- 안전한 답장
- 조금 더 다정한 답장
- 선을 긋는 답장

문장은 부담이 낮고 자연스러워야 한다.
지나치게 작위적이거나 기술적으로 보이는 문장은 피한다.

### 3. internal evaluation

내부 검수용 평가 정보다.
사용자에게 직접 노출하지 않는다.
실패 유형 코드 F01~F15를 붙일 수 있는 구조를 둔다.

## 중요 평가축

- 부담도
- 자연스러움
- 실제 사용 의향
- 관계 리스크
- 감정 과잉 여부
- 상대방 통제/조종 느낌 여부
- 사용자가 후회할 가능성

## 초기 MVP 범위

처음부터 완성형 앱을 만들지 않는다.
먼저 아주 단순한 MVP를 만든다.

MVP는 아래 기능만 포함한다.

1. 사용자가 현재 상황을 입력한다.
2. 상대방의 최근 메시지를 입력한다.
3. 사용자의 의도를 입력하거나 선택한다.
4. 앱이 strategy_summary를 보여준다.
5. 앱이 recommendations 3개를 보여준다.
6. internal evaluation 구조는 내부적으로만 유지한다.

## 아직 하지 않을 것

초기에는 아래 기능을 만들지 않는다.

- 로그인
- 회원가입
- 결제
- 복잡한 DB
- 배포
- 모바일 앱 전환
- 관리자 페이지
- 사용자 기록 저장
- 알림 기능
- 커뮤니티 기능
- 복잡한 AI 파인튜닝

## 초기 개발 원칙

초기에는 로컬에서 돌아가는 웹앱 구조를 목표로 한다.
단, 터미널 실행은 사용자의 명시 허가 전까지 금지한다.
