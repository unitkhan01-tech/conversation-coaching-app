# 서버 아키텍처 결정

## 1. 결정

대화코칭어플의 초기 AI 연결 구조는 Vercel 서버리스 함수 방식으로 진행한다.

현재 앱은 정적 웹앱이다.

현재 주요 앱 파일:

- index.html
- style.css
- app.js

이 구조에서는 AI API 키를 안전하게 숨길 수 없다.  
따라서 실제 AI API를 연결하려면 반드시 서버 또는 서버리스 함수가 필요하다.

## 2. 왜 서버리스 함수가 필요한가

AI API 키는 브라우저에서 보이는 코드에 들어가면 안 된다.

**금지 위치**

- index.html
- style.css
- app.js
- 브라우저 localStorage
- 브라우저 console
- 공개 문서 파일
- GitHub 공개 저장소

**허용 위치**

- 서버 환경 변수
- 서버리스 함수 환경 변수
- 배포 플랫폼의 Secret
- 배포 플랫폼의 Environment Variable

## 3. 선택한 구조

초기 구조는 아래와 같다.

사용자 브라우저  
→ app.js  
→ `/api/generate-reply`  
→ Vercel 서버리스 함수  
→ AI API  
→ 서버리스 함수에서 응답 검증  
→ app.js로 결과 반환  
→ 사용자 화면에 표시

## 4. 이 구조의 장점

Vercel 서버리스 함수 방식을 선택하는 이유는 아래와 같다.

1. 비개발자에게 서버 운영 부담이 적다.
2. API 키를 브라우저 코드에서 숨길 수 있다.
3. 프론트엔드와 API endpoint를 한 프로젝트에서 관리할 수 있다.
4. 초기 MVP에 충분하다.
5. 나중에 Next.js 또는 더 큰 구조로 확장하기 쉽다.
6. 로컬 프로토타입에서 실제 AI 연결 단계로 넘어가기 좋다.

## 5. 현재 단계에서 하지 않을 것

아래는 아직 하지 않는다.

- 실제 API 키 입력
- 실제 AI API 호출
- Vercel 배포
- npm install
- 프로젝트 구조 변환
- Next.js 전환
- 로그인
- 결제
- DB 저장
- 사용자 기록 저장

## 6. 다음 단계

다음 단계는 실제 구현이 아니라, 서버리스 함수에 필요한 파일 구조와 데이터 흐름을 설계하는 것이다.

그 후 사용자가 명시적으로 허락하면 실제 코드 파일을 생성한다.

## 7. 최종 원칙

AI 연결은 필요하다.

하지만 API 키 보안 구조 없이 AI 연결을 시작하면 안 된다.

따라서 이 프로젝트는 Vercel 서버리스 함수 구조를 먼저 설계한 뒤, 안전하게 AI API 연결을 진행한다.
