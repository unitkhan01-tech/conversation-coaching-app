# MVP_V02_IMPROVEMENT_SPEC.md

## 1. v0.2 목표

v0.2의 목표는 단순히 답장을 보여주는 앱이 아니라, 사용자가 자신의 대화 상황을 더 차분히 판단하고 덜 후회할 답장을 고를 수 있게 만드는 것이다.

v0.2는 아래 방향을 강화한다.

1. UI 신뢰감 개선
2. 코칭 구조 강화
3. 추천 답장 품질 개선
4. 사용자의 감정 과잉 방지
5. 상대방 조종이나 밀당 방지
6. 실제 카카오톡으로 보낼 수 있는 자연스러운 문장 강화

## 2. v0.2에서 해결해야 할 핵심 문제

### 문제 1. UI가 너무 기능 확인용이다

현재 화면은 입력창과 결과창만 있는 구조다.
사용자가 이 앱을 신뢰하거나, 감정적으로 안정감을 느끼기 어렵다.

v0.2에서는 아래를 개선한다.

- 첫 화면 문구 개선
- 입력 영역 안내 강화
- 결과 카드 디자인 개선
- 답장 후보 간 차이 명확화
- 모바일 사용성 개선
- 전체적으로 차분하고 안전한 분위기 강화

### 문제 2. 추천 대화가 얕다

현재 답장은 상황별 예시 문장 수준이다.
사용자가 “이거 진짜 보내도 되겠다”고 느끼기에는 부족하다.

v0.2에서는 아래를 개선한다.

- 답장 문장을 더 짧고 자연스럽게 만든다.
- 오글거리는 문장을 줄인다.
- 상담문 같은 문장을 줄인다.
- 실제 메신저 말투에 가깝게 만든다.
- 사용자의 의도와 상황에 따른 차이를 더 분명히 만든다.

### 문제 3. 코칭감이 부족하다

이 앱은 답장 생성기가 아니라 대화코칭 앱이어야 한다.

v0.2에서는 결과 영역을 아래처럼 개선한다.

1. 지금 상황 요약
2. 지금 조심할 점
3. 답장 방향
4. 답장 후보 3개
5. 각 답장을 쓰면 좋은 상황
6. 피해야 할 답장 방향

단, 화면이 너무 무거워지지 않도록 짧고 명확하게 보여준다.

## 3. v0.2 결과 구조 개선안

사용자에게 보여줄 결과는 아래 구조를 따른다.

### 1. 상황 정리

사용자가 놓치기 쉬운 맥락을 짧게 정리한다.

예:
상대가 바쁘다고 말한 상황이라, 지금은 답장을 강요하기보다 부담을 낮추는 쪽이 안전합니다.

### 2. 조심할 점

사용자가 감정적으로 과잉 반응하거나, 너무 차갑게 답하지 않도록 알려준다.

예:
서운함을 바로 크게 표현하면 상대가 방어적으로 느낄 수 있습니다.

### 3. 추천 방향

지금은 어떤 톤이 가장 안전한지 알려준다.

예:
가볍게 이해를 표현하고, 대화를 열어두는 답장이 좋습니다.

### 4. 답장 후보 3개

항상 아래 3개를 제공한다.

- 안전한 답장
- 조금 더 다정한 답장
- 선을 긋는 답장

### 5. 피해야 할 답장

사용자에게 직접 보내지 말아야 할 방향을 짧게 알려준다.

예:
“왜 답장이 늦어?”처럼 추궁하는 방식은 지금 상황에서는 부담을 줄 수 있습니다.

## 4. 추천 답장 품질 기준

v0.2 추천 답장은 아래 기준을 따른다.

- 실제 카카오톡으로 보낼 수 있어야 한다.
- 너무 길면 안 된다.
- 한 문장 또는 두 문장 안에서 끝나는 것이 좋다.
- 상대를 몰아붙이면 안 된다.
- 사용자를 지나치게 낮추면 안 된다.
- 일부러 질투를 유발하면 안 된다.
- 답장을 늦게 하라고 하면 안 된다.
- 상대를 시험하면 안 된다.
- 다정한 답장도 부담스럽지 않아야 한다.
- 선을 긋는 답장도 공격적이지 않아야 한다.

## 5. UI 개선 방향

v0.2 UI는 아래 원칙을 따른다.

- 더 넓고 안정적인 레이아웃
- 입력 영역과 결과 영역의 구분 강화
- 결과 카드의 가독성 개선
- 답장 후보 카드의 차이 명확화
- 복사 버튼은 유지
- 예시로 채우기와 테스트 상황 불러오기는 유지
- 불필요한 장식은 넣지 않음
- 전체 톤은 차분하고 신뢰감 있게 유지

## 6. v0.2에서 추가할 화면 요소

v0.2에서는 결과 영역에 아래 요소를 추가한다.

1. 상황 정리
2. 조심할 점
3. 추천 방향
4. 답장 후보
5. 피해야 할 답장

단, internal_evaluation은 계속 화면에 노출하지 않는다.

## 7. v0.2에서 하지 않을 것

아래는 아직 하지 않는다.

- AI API 연결
- 로그인
- 결제
- DB 저장
- 배포
- 서버 구현
- 모바일 앱 전환
- 복잡한 사용자 기록 기능
- 관리자 페이지

## 8. v0.2 작업 순서

v0.2는 아래 순서로 진행한다.

1. v0.1 불합격 사유 기록
2. v0.2 개선 명세 확정
3. DATA_STRUCTURE.md에 v0.2 출력 구조 반영
4. UI_WIREFRAME.md에 v0.2 화면 구조 반영
5. index.html, style.css, app.js 개선
6. 6개 테스트 시나리오 재검수
7. v0.2 통과 여부 판단

## 9. v0.2 통과 기준

v0.2는 아래를 만족해야 한다.

- UI가 기능 확인용 수준에서 벗어난다.
- 결과 영역에서 코칭감이 느껴진다.
- 답장 후보 3개가 명확히 구분된다.
- 최소 1개 답장은 실제로 보낼 만하다.
- 피해야 할 답장 방향이 표시된다.
- 조종, 밀당, 질투 유발, 상대 시험 문장이 나오지 않는다.
- internal_evaluation은 화면에 보이지 않는다.
- 사용자가 “왜 이 답장이 좋은지” 이해할 수 있다.

## 10. 최종 결론

v0.2의 핵심은 기능 추가가 아니라 품질 개선이다.

대화코칭어플은 답장을 많이 만들어주는 앱이 아니다.
사용자가 더 덜 후회하는 대화를 선택하게 돕는 앱이다.
