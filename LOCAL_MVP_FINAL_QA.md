# LOCAL_MVP_FINAL_QA.md

## 1. 목적

이 문서는 대화코칭어플 로컬 MVP가 실제 AI API 연결 전 단계에서 제품 철학, 화면 구조, 응답 품질 기준을 충족하는지 최종 점검하기 위한 문서다.

현재 단계의 목표는 완성형 서비스가 아니라, 아래가 제대로 작동하는지 확인하는 것이다.

1. 사용자가 상황을 입력할 수 있는가
2. 사용자가 상대방 메시지를 입력할 수 있는가
3. 사용자가 답장 의도를 선택할 수 있는가
4. strategy_summary가 표시되는가
5. recommendations 3개가 표시되는가
6. internal_evaluation이 화면에 노출되지 않는가
7. 답장이 조종, 밀당, 불안 증폭으로 흐르지 않는가

## 2. 현재 MVP 구성

현재 앱은 아래 3개 파일로 실행된다.

- index.html
- style.css
- app.js

현재는 정적 웹앱이다.
브라우저에서 index.html을 직접 열어 확인한다.

## 3. 현재 구현된 기능

현재 MVP에 구현된 기능은 아래와 같다.

- 현재 상황 입력
- 상대방 메시지 입력
- 답장 의도 선택
- 예시로 채우기
- 다시 쓰기
- 테스트 상황 불러오기
- 답장 방향 보기
- strategy_summary 출력
- recommendations 3개 출력
- 답장 복사
- 복사 완료 안내
- internal_evaluation 내부 구조 유지
- internal_evaluation 화면 미노출

## 4. 수동 테스트 방법

브라우저에서 index.html을 연 뒤 아래 순서로 테스트한다.

1. 테스트 상황 불러오기에서 시나리오를 선택한다.
2. 현재 상황, 상대방 메시지, 답장 의도가 자동 입력되는지 확인한다.
3. 답장 방향 보기 버튼을 누른다.
4. 상황 정리가 표시되는지 확인한다.
5. 답장 후보 3개가 표시되는지 확인한다.
6. 각 답장 후보가 서로 다른 방향인지 확인한다.
7. 이 답장 복사 버튼이 작동하는지 확인한다.
8. 다시 쓰기 버튼이 입력값과 결과를 초기화하는지 확인한다.

## 5. 필수 테스트 시나리오

아래 6개 시나리오를 모두 테스트한다.

### 1. 호감 탐색 / 부담 없이 이어가기

확인 기준:

- 상대의 바쁨을 인정하는가
- 답장을 강요하지 않는가
- 가볍고 자연스러운가
- 불안을 키우지 않는가

### 2. 호감 탐색 / 조금 더 다정하게 표현하기

확인 기준:

- 다정하지만 부담스럽지 않은가
- 호감이 약하게 드러나는가
- 상대를 몰아붙이지 않는가

### 3. 텐션 하락 / 잘 모르겠다

확인 기준:

- 사용자의 불안을 키우지 않는가
- 무리하게 대화를 이어가라고 하지 않는가
- 가장 안전하고 부담 낮은 방향을 제안하는가

### 4. 서운함·갈등 / 차분히 말하기

확인 기준:

- 서운함을 인정하는가
- 상대를 비난하지 않는가
- 사용자의 감정을 지나치게 낮추지 않는가
- 공격적이지 않은가

### 5. 선 긋기

확인 기준:

- 거절이나 거리두기가 분명한가
- 공격적이지 않은가
- 사용자가 계속 맞춰줘야 하는 구조를 줄이는가

### 6. 관계 애매함

확인 기준:

- 상대 의도를 단정하지 않는가
- 사용자의 혼란을 인정하는가
- 안전하고 부담 낮은 방향을 제안하는가

## 6. 절대 실패 기준

아래 중 하나라도 발생하면 현재 MVP는 수정이 필요하다.

- internal_evaluation이 화면에 보인다.
- failure_codes가 화면에 보인다.
- 답장이 일부러 질투를 유발하라고 말한다.
- 답장을 일부러 늦게 하라고 말한다.
- 상대를 시험하라고 말한다.
- 상대 마음을 단정한다.
- 사용자의 불안을 키운다.
- 답장 후보 3개가 거의 똑같다.
- 선을 긋는 답장이 공격적이다.
- 답장이 실제 카카오톡으로 보내기 너무 길다.

## 7. 현재 단계 판정 기준

아래를 모두 만족하면 로컬 MVP v0.1 통과로 본다.

- 6개 테스트 시나리오가 모두 작동한다.
- strategy_summary가 표시된다.
- recommendations 3개가 표시된다.
- 답장 복사 기능이 작동한다.
- 다시 쓰기 기능이 작동한다.
- internal_evaluation이 화면에 노출되지 않는다.
- 금지 문장 유형이 나오지 않는다.
- 사용자가 실제로 보낼 수 있는 답장이 최소 1개 이상 포함된다.
