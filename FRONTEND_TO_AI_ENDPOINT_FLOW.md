# 프론트엔드 → AI endpoint 데이터 흐름

## 1. 목적

이 문서는 프론트엔드에서 AI 서버리스 endpoint까지 데이터가 어떻게 이동해야 하는지 설명한다.

목표는 AI API 키를 숨기면서도, 사용자가 입력한 상황을 기반으로 관계 목표형 답장 후보를 받는 것이다.

## 2. 전체 흐름

전체 흐름은 아래와 같다.

1. 사용자가 현재 상황을 입력한다.
2. 사용자가 상대방 메시지를 입력한다.
3. 사용자가 관계 목표를 선택한다.
4. app.js가 입력값을 수집한다.
5. app.js가 `/api/generate-reply`로 POST 요청을 보낸다.
6. 서버리스 함수가 요청을 받는다.
7. 서버리스 함수가 입력값을 검증한다.
8. 서버리스 함수가 AI API를 호출한다.
9. AI가 관계 목표형 답장 후보를 생성한다.
10. 서버리스 함수가 AI 응답 구조를 검증한다.
11. 서버리스 함수가 사용자에게 보여줄 필드만 반환한다.
12. app.js가 결과를 화면에 렌더링한다.

## 3. 프론트엔드 요청 예시

app.js가 서버로 보낼 데이터는 아래와 같다.

```json
{
  "situation": "내가 상대에게 톡을 몇 번 보냈는데 상대가 부담스러워하는 것 같아요.",
  "other_message": "톡을 왜 이렇게 많이 해!",
  "user_goal": "내가 너무 매달려 보이고 싶지 않다"
}
```

## 4. 서버리스 함수가 해야 할 입력 검증

서버리스 함수는 아래를 확인한다.

1. situation이 문자열인지
2. other_message가 문자열인지
3. user_goal이 문자열인지
4. 입력값이 너무 길지 않은지
5. 빈 값이 아닌지
6. 불필요한 개인정보가 포함되어 있지 않은지

## 5. AI에게 전달할 핵심 지시

AI는 아래 기준을 따라야 한다.

- 성숙한 모범답안처럼 쓰지 않는다.
- 상담문처럼 쓰지 않는다.
- 실제 카카오톡 말투에 가깝게 쓴다.
- 상대 메시지의 기능을 먼저 읽는다.
- 사용자의 관계 목표를 반영한다.
- 답장 후보 3개는 서로 다른 목적을 가져야 한다.
- 상대를 조종하거나 일부러 불안하게 만들지 않는다.
- 사용자가 너무 휘말리지 않도록 돕는다.

## 6. 서버리스 함수 응답 예시

서버리스 함수는 프론트엔드에 아래 구조만 반환한다.

```json
{
  "relationship_read": {
    "current_dynamic": "상대가 연락 양에 부담을 표현한 상황입니다."
  },
  "reply_options": [

      "goal": "maintain_flow",

      "message": "아 미안 ㅋㅋ 좀 많았나",

    },
    {
      "goal": "regain_leverage",
      "label": "주도권 회복",
      "message": "오키 좀 줄일게",
      "intended_effect": "길게 해명하지 않고 내 페이스로 물러납니다."
    },
    {
      "goal": "set_boundary",

  "before_send_check": "길게 사과하면 더 쫓아가는 느낌이 날 수 있습니다."

```

## 7. 프론트엔드 렌더링 원칙

프론트엔드는 아래만 보여준다.

- 지금 관계 흐름
- 답장 후보 3개
- 각 답장의 노리는 흐름
- 보내기 전 체크
- 복사 버튼

프론트엔드는 아래를 보여주지 않는다.

- internal_evaluation
- failure_codes
- API 키
- 프롬프트 원문
- AI 응답 전체 원문
- 개발용 로그

## 8. 오류 처리

서버 요청이 실패하면 아래 문구를 보여준다.

지금은 답장 후보를 불러오지 못했어요. 잠시 후 다시 시도해 주세요.

가능하면 현재 로컬 fallback 답장 후보를 보여준다.

## 9. 다음 구현 단계

다음 실제 구현 단계는 아래 순서로 진행한다.

1. package.json 생성
2. api/generate-reply.js 생성
3. .env.example 생성
4. vercel.json 생성 여부 검토
5. app.js에서 fetch 요청 구조 추가
6. fallback 유지
7. 실제 API 키는 코드에 넣지 않고 환경변수로 설정
8. 로컬 또는 Vercel에서 테스트

## 10. 최종 원칙

프론트엔드는 사용자 경험을 담당한다.

서버리스 함수는 AI 호출과 보안을 담당한다.

API 키는 절대 브라우저 코드에 들어가면 안 된다.
